## 사실 (Fact)
/* 초기 목표와 달라진 점이 있나요? 프로젝트는 어느 정도 완성이 되었나요? 프로젝트를 진행하며 경험한 이슈 중 세 가지를 꼽고, 이를 해결한 방법을 알려주세요. 문제를 잘 해결하는 사람이라는 것을 보여주세요. */

+ 초기 목표와 달라진 점 
  + 초기에는 기본적인 기능만 달성하려고 했는데 하다보니 필요한 기능이 많아져서 기획이 바뀌게 되었다.
  + 관리자용 계정을 만들었다.(상품 수정, 삭제, 열람)
  + 바로구매기능을 추가하였다.
  + 상품을 최신순, 가격순, 인기순으로 정렬하는 기능을 추가하였다.
  + 검색기능을 추가하였다.
  + 찜하기 기능을 추가하였다.
  + 마이페이지를 추가하여 찜한 상품과 그동안 주문했던 주문사항을 열람할 수 있게 하였다. 

+ 프로젝트 완성도
  + 초기에 기획했던 것은 거의 달성하였지만, 하면서 추가적으로 생긴 목표는 완벽하게 달성하지 못했다. 70% 정도 달성한 것 같다. 

+ 프로젝트를 진행하면서 경험한 이슈 
  1. 장바구니에 담긴 상품의 렌더링 속도가 느렸다. 
    + 처음에는 bags에 있는 해당 사용자의 장바구니 리스트를 통신하고 그에 해당하는 items를 다시 한 상품 당 두 번 통신을 해줘야 했다.
    + expand 를 사용해서 bags에 있는 아이템의 정보를 통신 한 번에 불러올 수 있게 되어 속도가 향상되었다. 

  2. 장바구니에 담을 때 사용자의 장바구니에 이미 있는 상품인지 확인하는 기능
    + 처음에는 통신을 통해서 사용자의 장바구니를 받고 직접 filter메소드를 사용해서 장바구니에 있는 상품인지 체크하였다.
    + json-server 문서를 읽어보니 통신을 할 때 부터 사용자의 장바구니에 상품이 있는지 한 번에 확인 할 수 있다는 것을 알게 되었다.(& 사용)

  3. 최신순, 가격순으로 정렬할 때 가격이 문자열로 저장되서 제대로 정렬되지 않았다.
    + 기능을 만들어 놓고 제대로 확인하지 않았다.
    + 계속 모르고 있다가 뒤늦게 발견하였다.
    + 가격을 number타입으로 저장하였다.

  4. css를 잘 다루지 못해서 막막했다. 
    + 프로젝트를 하면서 css가 가장 걱정이 되었다.
    + 기본적인 정렬만 하자 라는 마음으로 이것저것 공부해서 스타일링을 하였다. bulma를 사용해서 그래도 조금은 괜찮았던 것 같다. 
    + 그래도 css에 대한 두려움이 조금은 줄어든 것 같다. css 공부를 해야될 것 같다. 

## 긍정적인 변화와 느낀 점 (Feeling & Finding)
/* 결과보다는 과정이 중요합니다. 지난 번 해커톤과 비교해봤을 때, 프로젝트를 진행하면서 긍정적으로 바뀐 부분은 무엇인가요? 프로젝트를 진행하며 느꼈던 감정의 파도는 무엇이었나요? 프로젝트가 완성된 지금, 그 감정을 돌아보는 시간을 가져 주세요. 또, 프로젝트를 진행하면서 새롭게 알게된 사실을 적어주세요.  */ 

사실 저번 해커톤 때는 내가 한 일이 별로 없었다. 결과물은 너무 좋았고 옆에서 배운 것도 많았지만 기여한 정도가 적다고 생각하다보니 내가 한 프로젝트는 아니라는 생각을 했었다. 
이번 프로젝트로 그런 부분이 조금 해소된 것 같다. 많은 기능을 구현한 것도 아니고 화면이 아주 만족스럽지도 않지만, 초기에 목표했던 기본적인 기능을 정해진 기간안에 끝내겠다는 목표는 이뤘다.  

프로젝트를 하면서 가장 좋았던 것은 일주일동안 계속해서 프로젝트만 생각했던 시간인 것 같다. 하루종일 몰입해서 코딩했던 시간들이 재미있었다.   
목표한 기능을 하나하나 할 때마다 대단한 건 아니지만, 성취감을 느낄 수 있었다. 그리고 더 열심히 공부하고 더 많은 것을 알면 더 좋은 코드를 짤 수 있구나 라고 체감했다. 뭔가 더 간결하고 체계적으로 짤 수 있는 방법이 있는데 내가 잘 못하고 있는 것 같았다. 아직 공부해야 할 것이 너무 너무 많다는 것을 느꼈다. 좀 더 열심히 공부를 해야 할 동기가 생겼다. 

초기 기획의 중요성을 알게되었다. 뒤로 갈수록 추가적인 기능이 많이 생겼고, 템플릿을 재사용하고 이미 짜놓은 함수를 재사용하고 싶어도 아주 조금씩 달라서 다시 코드를 바꿔야 하는 상황이 생겼다. 그래서 중복되는 코드가 생겼다.
이번 프로젝트에서 부족한 점을 많이 느꼈으니, 다음 프로젝트는 좀 더 보완해서 개선할 수 있을 것 같다.

## 개선할 점과 앞으로 해볼 일 (Future Action)
/* 최종 프로젝트 때 실천해보고 싶은 개선점이 있나요? 구체적인 액션 플랜을 적어 주세요. */

+ 템플릿을 사용하였는데 화면 단위로 템플릿을 쓰다보니 중복되는 부분이 많았다. 템플릿을 잘게 쪼개서 재사용하기 쉽게 만들고 싶다.

+ class 이름이나, 함수이름, 변수이름을 누구나 알아보기 쉽게, 사용하기 쉽게 개선하고 싶다. 지금은 너무 되는대로 막 지어놓은 느낌이다.

+ 좀 더 알아보기 쉽고 재사용하기 쉽게 함수로 정리하고 싶었는데 조금밖에 개선하지 못하였다.  

+ 일단 되게만 하자 라는 마음가짐으로 시작했더니 너무 지저분한 코드를 짜게 된 것 같다. 중간중간 점검하는 시간도 필요한 것 같다. 

+ 초기에 자세한 기획을 해야 겠다. 이번 프로젝트에서는 너무 간단하게 기획을 하고 시작해서 끝으로 갈수록 덕지덕지 붙여놓은 느낌이다.

+ 상품의 옵션을 설정하지 않고 시작했다. 의류 쇼핑몰인데 사이즈가 없다... 사이즈, 색상의 옵션을 추가하고 싶다.

+ 관리자가 상품을 좀 더 편하게 관리할 수 있도록 재고를 설정하고 재고 관리를 할 수 있도록 하고 싶다. 

+ 상품을 검색할 때 지금은 검색한 문자의 전체 문자열이 있는지 없는지만 확인하고 있다. 단어의 순서를 반대로 입력하면      찾을 수 없다.
  검색기능을 고민하다가 일단은 간단하게만 구현해놓았는데 최종 프로젝트에는 좀 더 사용자가 제대로 상품의 이름을 몰라도 검색할 수 있도록 하고 싶다. 
 
+ 화면이 너무 단순하다. 나의 미적감각으론 괜찮은데 다른사람이 보면 좀 심할 것 같다...  css를 좀 더 공부해야겠다.
